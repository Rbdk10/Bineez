Bineez Style Guide

Design system for Rankings, Brackets, and Matchups

Introduction

The Bineez design system emphasizes clarity, competitiveness, and community. This style guide provides comprehensive specifications for implementing the brand across all digital touchpoints. The palette combines deep navy blues with energetic accents to create a trustworthy, dynamic look suited to rankings, custom brackets, and head-to-head comparisons.

Brand Values

  • Clarity: Clear hierarchy for matchups and progression
  • Trust: Reliable, accessible UI for fair rankings
  • Energy: Accents that guide action without distraction
  • Community: Inclusive design for shared participation

Layout Guidelines

Container Widths

  • Maximum Content Width: 1200px for main content
  • Form Width: Maximum 600px for optimal readability
  • Text Column: Maximum 65-75 characters per line (roughly 700px)

Grid System

Use a 12-column grid system with 20-30px gutters. This provides flexibility for various layouts while maintaining consistency.

/* Responsive Grid Example */ .grid { display: grid; grid-template-columns: repeat(12, 1fr); gap: 24px; } /* Responsive Breakpoints */ Mobile: 320px - 767px Tablet: 768px - 1023px Desktop: 1024px+

White Space

Generous white space is crucial for a professional, uncluttered appearance:

  • Maintain minimum 40px padding on page sections
  • Use 24px spacing between major elements
  • Keep 16px minimum margin around text blocks
  • Don't be afraid of empty space - it enhances focus

Accessibility Standards

WCAG Compliance: All components must meet WCAG 2.1 Level AA standards:
  • Minimum 4.5:1 contrast ratio for normal text
  • Minimum 3:1 contrast ratio for large text (18pt+)
  • All interactive elements must be keyboard accessible
  • Focus states must be clearly visible
  • Color must not be the only means of conveying information
